# Saudi Market Radar — رادار السوق السعودي

نسخة أولية محلية قابلة للتشغيل تجمع أخبار الاقتصاد والشركات السعودية من RSS، تنظف النص، تزيل الأخبار المتكررة، تصنفها قطاعيًا، وتعرضها في لوحة عربية RTL. لا تتضمن اللوحة توصيات استثمارية أو توقعات لأسعار الأسهم.

## المزايا

- Next.js مع TypeScript وApp Router.
- SQLite محلي عبر `node:sqlite`، من دون خدمة قاعدة بيانات خارجية.
- مصادر RSS قابلة للتعديل من `config/sources.ts`.
- معالجة محلية للنص تشمل التنظيف والتصنيف واحتساب الأهمية والتشابه النصي.
- بيانات تجريبية تلقائية، وفلاتر للقطاع والمصدر والتاريخ.
- موجز لأهم 5 تطورات وقسم للفرص والمؤشرات المعلوماتية.
- واجهة API محلية لجلب الأخبار وتحديثها.
- اختبارات أساسية للتصنيف وإزالة التكرار.

## المتطلبات والتشغيل

يتطلب Node.js 22.5 أو أحدث (ويُنصح بالإصدار 24 أو أحدث) وnpm.

```bash
npm install
npm run dev
```

ثم افتح `http://localhost:3000`. وللتحقق الكامل:

```bash
npm run lint
npm test
npm run build
npm start
```

## إعداد المصادر

عدّل القائمة في `config/sources.ts`. لكل مصدر معرّف، واسم ظاهر، ورابط RSS، وحالة تفعيل، ولغة. فشل مصدر واحد لا يمنع معالجة بقية المصادر.

```ts
{
  id: "example",
  name: "اسم المصدر",
  url: "https://example.com/rss.xml",
  enabled: true,
  language: "ar",
}
```

## قاعدة البيانات

ينشأ `data/radar.db` تلقائيًا عند أول تشغيل ويُستبعد من Git. عندما تكون القاعدة فارغة، تُضاف بيانات تجريبية من `lib/seed.ts`. لإعادة البدء، أوقف الخادم واحذف ملف القاعدة يدويًا.

## بنية المشروع

- `app/`: الصفحات ومسار API.
- `components/dashboard.tsx`: اللوحة والفلاتر.
- `config/sources.ts`: إعداد المصادر.
- `lib/db.ts`: مخطط SQLite والاستعلامات.
- `lib/rss.ts`: جلب RSS وتحليله.
- `lib/pipeline.ts`: الجمع وإزالة التكرار والحفظ.
- `lib/text.ts`: تنظيف النص والتشابه المحلي.
- `lib/classifier.ts`: التصنيف والأهمية وأثر الأعمال.
- `lib/summarizer.ts`: الملخص المحلي.
- `tests/`: الاختبارات الأساسية.

## حدود النسخة الأولية

- الملخص العربي استخراجي وقائم على قواعد. المحتوى الإنجليزي لا يُترجم آليًا لأن المشروع لا يستخدم خدمة مدفوعة.
- محلل RSS خفيف ويدعم RSS/Atom الشائع، وقد يحتاج موائمًا خاصًا لتنسيق غير قياسي.
- درجة الأهمية والمؤشرات تقديرية بقواعد شفافة، وليست رأيًا استثماريًا.
- التحديث يدوي عبر الزر، ولا توجد جدولة خلفية بعد.

## API المحلي

- `GET /api/news`: يعيد الأخبار المحفوظة.
- `POST /api/news`: يجلب المصادر المفعلة، يعالجها، ويحفظ الجديد.
